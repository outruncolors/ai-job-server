Implement Phase D1.4 of the Blaboratory vector-retrieval system. Branch
blaboratory-part2. Goal: replace the mechanical recency gather in the [You Know]
context section with HYBRID retrieval (recent floor ∪ relevant top-k), and run
the indexer once per tick. Mechanical recency stays as the fallback.

Read first:

1. context_pipeline:
   - _build_query(resident) -> str: the resident's recency-window items joined
     ("what's on my mind right now").

     apply_caps. If not VEC_AVAILABLE or embed fails → return the mechanical gather
     unchanged.
   - wire build_context's [You Know] to retrieve_memories.
2. tick_runner.run_tick: call memory_index.index_pending() once before the
   occupant loop.

Done when tests/apps/test_context_pipeline.py (extended) passes, with a fake index:

  alone would have dropped;

- REGRESSION GUARD: with the index unavailable, build_context output is
  byte-identical to today's mechanical-gather output.
Full existing suite stays green.

Env: .venv/bin/python, .venv/bin/pytest (asyncio_mode=auto), py_compile.
